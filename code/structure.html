Code: Structure

The program is divided into multiple files. In red are files that you usually wouldn't modify.
It is important to distinguish the "helper" classes (PackMLStateModel.cs, Unit.cs, and Process.cs) from the "real" classes used in the OPCUA server (PackMLStateModelType, UnitType, and ProcessType)

File name Description
Program.cs Starts the server.
TestServerManager.cs Defines actions to take once the server is started.
PackMLStateMachineNodeManager.cs Manages the nodes. Modify the overriden Startup() method to create instances of a class.
PackMLStateMachineClasses.cs Defines the two classes (PackMLStateModel and UnitModel) that can be used to create instances in a OPCUA client.
PackMLStateMachineIdentifier.cs Link all the objects to a node and defines constants. (Such as browse name, URI namespace...)
PackMLStateMachineTypes.cs Define all the different types.
PackMLStateModel.cs Helper class that models a state machine. Parent class of Unit.cs
PackMLStateModelTypeMethods.cs Defines the implementation of the methods of the PackMLStateModelType class.
Unit.cs Helper class that defines a unit/a machine. Child class of PackMLStateModel.cs
UnitTypeMethods.cs Defines the implementation of the methods of the UnitType class.
Process.cs Helper class that defines a process.
DBManager.cs Class to control the database.
