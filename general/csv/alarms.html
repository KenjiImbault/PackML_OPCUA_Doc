Alarm csv file

In the "Data" folder, you can find an example on how to format the .csv files for the alarms called "_Template_Alarms.csv"
In more detail (Please note that the first line is reserved for headers):

  • The first column contains the alarm ID of each alarm.
    Each ID is unique. (However it must only be unique in the same file. Meaning you can have one ID "1" in file A and one ID "1" in file B)
  • The second column contains the transition tag value to apply once the alarm is triggered. Please refer to this link to see the correspondance between the transitions and their tag value.
  • The third column contains the message to display once the alarm is triggered.
Reading this file using a method in an instance will add all of the alarms in the instance.
