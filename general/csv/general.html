General csv file

In the "Data" folder, you can find an example on how to format the .csv files for the alarms called "__Template_Machine.csv"
In more detail (Please note that the first column is reserved for headers):

  • General
    • The cell B1 is where you will input the name of the machine.
    • The cell B2 is where you will input the state of the machine. (In all caps, using the name of the state.)
  • Alarms
    • The cells on the 4th line will be where you input the ID of the alarms.
    • The cells on the 5th line will be where you input the transition of the alarms. (Make sure that the transition is written exactly like in the cheatsheet.)
    • The cells on the 6th line will be where you input the message of the alarm.
    • The cells on the 7th line will be where you input the state of the alarm. ("TRUE" for ON, or "FALSE" for OFF)
  • Stack lights
    • The cells on the 9th line will be where you input the ID of the stack lights.
    • The cells on the 10th line will be where you input the description of the stack lights.
    • The cells on the 11th line will be where you input the state of the stack lights. ("TRUE" for ON, or "FALSE" for OFF)
  • Buttons
    • The cells on the 13th line will be where you input the ID of the buttons. If a button has multiple transitions, type the ID multiple times.
    • The cells on the 14th line will be where you input the names of the buttons. If a button has multiple transitions, type the name multiple times.
    • The cells on the 15th line will be where you input the transitions of the buttons. (Make sure that the transition is written exactly like in the cheatsheet.)
Reading this file using a method in an instance will:
  • Change the name of the machine
  • Change the state of the machine
  • Add all of the alarms
  • Add all of the stack lights
  • Add all of the buttons
